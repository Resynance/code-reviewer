# REST API Reference

Base URL: `http://localhost:1500`. All bodies and responses are JSON. Errors use
FastAPI's `{"detail": "..."}` shape with the noted status code.

> Interactive docs are auto-generated by FastAPI at **`/docs`** (Swagger UI)
> and **`/redoc`** ŌĆö available in local development only. In production
> (Vercel) all three schema endpoints (`/docs`, `/redoc`, `/openapi.json`) are
> disabled to avoid exposing the API surface to unauthenticated callers.

## Reviews

### `POST /api/review` ŌĆö enqueue
Reviews run **asynchronously** so a slow model call doesn't hold one request open
past the serverless function limit. This endpoint enqueues a job and returns its id.

Request:
```json
{
  "pr_number": 201,
  "repo": "my-org/my-repo",
  "title": "Add JWT validation",
  "description": "",
  "diff": "diff --git a/... ",
  "author": "dev",
  "base_branch": "main",
  "files_changed": ["src/auth/middleware.py"]
}
```
Only `pr_number`, `repo`, `title`, and `diff` are required. Returns
`{ "id": "<uuid>", "status": "queued" }`. `400` if `OPENROUTER_API_KEY` is not set.

The client then calls `POST /api/review/{id}/run` and polls `GET /api/review/{id}`.

### `POST /api/review/{id}/run`
Execute a queued job (runs the review, persists the outcome to the job). Idempotent
ŌĆö a job already running/finished is returned as-is. Returns the job (see below).

### `GET /api/review/{id}`
Poll a job. Returns `{ id, status, request, result, error, created_at, updated_at }`
where `status` Ōłł `queued|running|done|error`. When `done`, `result` is:
```json
{
  "pr_number": 201,
  "summary": "ŌĆ”",
  "approved": false,
  "confidence": 0.82,
  "issues": [
    { "severity": "high", "file": "ŌĆ”", "description": "ŌĆ”", "suggestion": "ŌĆ”", "past_decision_ref": "PR #142" }
  ],
  "suggestions": [
    { "type": "security", "description": "ŌĆ”", "past_decision_ref": null }
  ],
  "past_decisions_applied": [
    { "ref": "PR #142", "summary": "ŌĆ”", "how_applied": "ŌĆ”" }
  ]
}
```
`issue.severity` Ōłł `critical|high|medium|low`. `suggestion.type` Ōłł
`security|performance|architecture|style|test_coverage`. `404` for an unknown id.

Each finished review is also **saved to history** (best-effort ŌĆö a save failure
never fails the review).

### `GET /api/reviews?repo=&pr_number=&limit=50`
Saved review runs, newest first (full history ŌĆö every run is kept, including
re-reviews of the same PR). All query params optional. `limit` defaults to 50,
max 500 (`422` if exceeded).
```json
{ "reviews": [
  { "id": 12, "repo": "org/a", "pr_number": 88, "title": "ŌĆ”", "author": "dev",
    "approved": false, "confidence": 0.82, "summary": "ŌĆ”",
    "issues": [...], "suggestions": [...], "past_decisions": [...],
    "source": "api", "created_at": "2026-06-18TŌĆ”" }
], "count": 1 }
```
`source` is `api` (UI) or `webhook` (GitHub PR event).

### `POST /api/pr-comment`
Post selected review findings as a comment on the PR.
```json
{ "repo": "org/a", "pr_number": 88, "body": "## ­¤ż¢ ReviewBot\n**Issues**\n- ŌĆ”" }
```
Returns `{ "html_url": "https://github.com/org/a/pull/88#issuecomment-ŌĆ”" }`. The
GitHub token needs **write** access (`repo` / `pull_requests:write`); `400`
without a token or empty body, `502` on GitHub errors (incl. a permissions hint).

### `POST /api/issue`
Open a new GitHub issue from selected review findings.
```json
{ "repo": "org/a", "title": "[HIGH] auth.py: missing token expiry check", "body": "## ­¤ż¢ ReviewBot\nŌĆ”" }
```
Returns `{ "html_url": "https://github.com/org/a/issues/9" }`. The GitHub token
needs **write** access (`repo` / `issues:write`); `400` without a token or empty
title, `502` on GitHub errors (incl. a permissions hint).

## Decisions

### `GET /api/decisions?k=20&repo=`
List recent decisions. `repo` is optional: a repo name, `*` for globals only, or
omitted for all. `k` defaults to 20, max 200 (`422` if exceeded).
```json
{ "decisions": [ { "doc_id": "ŌĆ”", "ref": "ŌĆ”", "summary": "ŌĆ”", "reasoning": "ŌĆ”",
                   "outcome": "ŌĆ”", "date": "ŌĆ”", "repo": "org/a", "score": 0.91 } ],
  "count": 1 }
```

### `POST /api/decisions/search`
Semantic search.
```json
{ "query": "auth token validation", "k": 10, "repo": "org/a" }
```
`repo` optional (same semantics as above). `k` defaults to 10, max 100 (`422` if
exceeded). Returns `{ "results": [...], "count": N }`.

### `POST /api/decisions`
Manually add a decision (e.g. an ADR).
```json
{ "ref": "ADR-007", "summary": "Use JWT", "reasoning": "Stateless",
  "outcome": "approved_and_merged", "metadata": { "repo": "*" } }
```
Set `metadata.repo` to a repo name, or `*` for a global decision. Returns
`{ "doc_id": "ŌĆ”", "ref": "ADR-007" }`.

### `DELETE /api/decisions/{doc_id}`
Removes a decision. Returns `{ "deleted": "<doc_id>" }`.

## Settings

### `GET /api/settings`
Secrets are reported as booleans, never echoed.
```json
{ "repos": ["org/a"], "github_token_set": true, "webhook_secret_set": false,
  "openrouter_model": "anthropic/claude-sonnet-4.5", "openrouter_provider": "" }
```

### `PUT /api/settings`
Update settings. **Only provided (non-null) fields change**; send `""` to clear a
value (falls back to env/default), omit/null to leave unchanged.
```json
{ "github_token": "ghp_ŌĆ”", "webhook_secret": "ŌĆ”", "repos": ["org/a", "org/b"],
  "openrouter_model": "openai/gpt-4o", "openrouter_provider": "Azure" }
```
Returns the same shape as `GET /api/settings`.

## Repositories

### `GET /api/repos`
`{ "repos": ["org/a", "org/b"] }`

### `POST /api/repos`
`{ "repo": "org/a" }` ŌåÆ `{ "repos": [...] }`. `400` if not in `owner/repo` form.

### `DELETE /api/repos?repo=org/a`
Removes a repo. Returns `{ "repos": [...] }`.

### `GET /api/github/owners`
Owner accounts the configured token can browse ŌĆö the authenticated user plus
their orgs. Used to discover repos instead of typing them.
```json
{ "owners": [ { "login": "you", "type": "user" }, { "login": "acme", "type": "org" } ] }
```
`400` without a token; `502` on GitHub errors (the message hints at needing
`read:org`/`repo` scope + SSO authorization).

### `GET /api/github/repos?owner=acme&type=org`
Repos under an owner the token can access (newest-updated first). `type` is
`org` (`/orgs/{owner}/repos`) or `user` (the authenticated user's own repos).
```json
{ "owner": "acme", "repos": [ { "full_name": "acme/api", "name": "api", "private": true, "description": "ŌĆ”" } ] }
```

## Access allowlist

Who may sign in and use the app, stored in the `access_allowlist` table and
editable at runtime (no redeploy; changes apply within ~30s). The `ALLOWED_EMAILS`
env var remains an optional bootstrap/fallback.

**Admin requirement:** `POST` and `DELETE` are restricted to the configured admin.
The admin is resolved from the `ADMIN_EMAIL` env var (explicit), falling back to
the first entry in `ALLOWED_EMAILS`. When `ALLOWED_EMAILS=*` and no `ADMIN_EMAIL`
is set, these endpoints return `403` ŌĆö manage the allowlist via env var directly
in that case. `GET` is readable by any authenticated user.

### `GET /api/access` ŌåÆ `{ "emails": ["you@co.com"] }`
### `POST /api/access` ŌĆö `{ "email": "user@co.com" }` ŌåÆ `{ "emails": [...] }` (`400` on invalid email, `403` if not admin)
### `DELETE /api/access?email=user@co.com` ŌåÆ `{ "emails": [...] }` (`403` if not admin)

### `POST /api/backfill`
Import a repo's closed PRs into the decision store (server-side).
```json
{ "repo": "org/a", "pages": 5 }
```
Returns `{ "repo": "org/a", "imported": 42 }`. `400` if no GitHub token is
configured or the repo is malformed; `502` on GitHub API errors (the message is
actionable, e.g. a 404 hint about private-repo access).

> Runs synchronously ŌĆö a large repo holds the request open until done.

### `GET /api/repos/open-prs?repo=org/a`
List a repo's **open** PRs that are **not yet in the decision store** (i.e. not
backfilled). Read-only discovery ŌĆö does not fetch diffs or run reviews.
```json
{
  "repo": "org/a",
  "open_pr_count": 12,
  "new_prs": [
    { "number": 88, "title": "Add caching", "author": "dev",
      "url": "https://github.com/org/a/pull/88", "created_at": "2026-06-01TŌĆ”", "draft": false }
  ]
}
```
`open_pr_count` is the total open PRs seen; `new_prs` are those whose decision id
(`<owner>-<repo>-pr-<number>`) isn't already stored. `400` if no GitHub token is
configured; `502` on GitHub API errors.

### `GET /api/repos/prs?repo=org/a`
List a repo's PRs for the Review page picker ŌĆö **open PRs first**, then closed,
each group newest-updated first.
```json
{ "repo": "org/a", "prs": [
  { "number": 88, "title": "Add caching", "author": "dev", "url": "ŌĆ”",
    "state": "open", "draft": false, "created_at": "ŌĆ”", "updated_at": "ŌĆ”" }
] }
```
`400` without a token; `502` on GitHub errors.

### `GET /api/repos/pr?repo=org/a&number=88`
Fetch one PR's metadata + unified diff + changed files, shaped for the review form.
```json
{ "pr_number": 88, "repo": "org/a", "title": "Add caching", "description": "ŌĆ”",
  "author": "dev", "base_branch": "main", "diff": "diff --git ŌĆ”",
  "files_changed": ["src/cache.py"] }
```
`400` without a token; `502` on GitHub errors.

## GitHub webhook

### `POST /webhook/github`
Triggers an automatic review on pull-request events.

- Verifies `X-Hub-Signature-256` (HMAC-SHA256) against the configured webhook
  secret. Returns `401` on a bad/absent signature, or if no secret is configured.
- `X-GitHub-Event: ping` ŌåÆ `{ "status": "pong" }`.
- Non-`pull_request` events ŌåÆ `{ "status": "ignored", "event": "ŌĆ”" }`.
- `pull_request` with action `opened|synchronize|reopened` ŌåÆ fetches the diff and
  reviews it, returning `{ "pr_number", "approved", "confidence", "issue_count" }`.

Configure in GitHub repo settings ŌåÆ Webhooks:

| Field | Value |
|---|---|
| Payload URL | `https://your-server/webhook/github` |
| Content type | `application/json` |
| Secret | the webhook secret set under Settings ŌåÆ GitHub Access |
| Events | Pull requests |
